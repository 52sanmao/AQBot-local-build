# 组件货架式入口配置设计

## 目标

把“标题栏配置”和“设置侧边栏配置”从“开关 + 上下箭头”改成统一的组件货架式编辑器。

用户可以：

- 从“可加入组件”里选择项目
- 一键加入到标题栏或侧边栏
- 在已加入区域里拖动排序
- 从标题栏中加入设置页入口，点击后直达对应设置页

## 范围

本次只改配置编辑体验和对应渲染逻辑，不扩展任意自定义动作。

- 标题栏来源：
  - 现有内置标题栏动作
  - 正式注册的设置页入口
- 侧边栏来源：
  - 正式注册的设置页入口

## 交互方向

显示设置页新增两个统一编辑器：

1. 标题栏组件货架
2. 设置侧边栏组件货架

每个编辑器分为两块：

- 可加入组件区
- 已显示组件区

支持操作：

- 点击选择组件
- “加入标题栏 / 加入侧边栏”
- 从已显示区移除
- 在已显示区拖动排序

## 数据模型

保留原有设置持久化入口，但把标题栏配置从“只支持内置动作”扩展为“支持两类项”：

- `builtin-action`
- `settings-section`

侧边栏配置仍然只保存设置页入口，但会在 UI 层改成货架式编辑。

## 渲染规则

### 标题栏

- 按配置顺序渲染
- 内置动作沿用现有按钮样式
- 设置页入口渲染为文字型快捷按钮
- 点击设置页入口时：
  - 若当前不在设置页，先进入设置页
  - 再切换到对应 section

### 设置侧边栏

- 继续渲染正式注册的设置页入口
- 顺序完全跟随配置
- 不显示未加入项

## 测试重点

- 标题栏能显示设置页快捷入口
- 点击标题栏设置页入口能切到目标 section
- 侧边栏按新配置顺序显示
- 货架编辑器能正确生成保存配置
